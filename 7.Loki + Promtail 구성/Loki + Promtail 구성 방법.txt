2. 실무형 Loki + Promtail 구성 방법

# 저장소 추가

1. helm repo add grafana https://grafana.github.io/helm-charts
2. helm repo update

# 커스텀 설정 파일 생성

1. cat << 'EOF' > loki-values.yaml
loki:
  persistence:
    enabled: true
    size: 50Gi      # 💡 앞서 만든 local-path 스토리지에서 50GB를 할당받습니다.
  config:
    limits_config:
      retention_period: 14d  # 💡 팁 적용: 로그 보관 기간을 14일로 제한
    compactor:
      working_directory: /data/loki/boltdb-shipper-compactor
      shared_store: filesystem
      retention_enabled: true
promtail:
  enabled: true     # 💡 수집기(Promtail)를 모든 노드에 배포합니다.
EOF

# Helm 배포

1. helm install loki-stack grafana/loki-stack \
  --namespace monitoring \
  -f loki-values.yaml

# 파드 기동 확인

1. kubectl get pods -n monitoring | grep -E "loki|promtail"

---

# Grafana 로그 실시간 확인하기

1. "Data sources" 검색 -> Add new data source 클릭 -> Loki 선택.
2. Connection 값 http://loki-stack.monitoring:3100 입력 -> Save & test 클릭.
-> 초록색 "Data source connected and labels found" 메시지가 뜨면 완벽, Pod, svc 전부 Running인데 빨간색이 뜬다면 Log 확인.
-> kubectl logs -n monitoring loki-stack-0, kubectl logs -n monitoring -l app.kubernetes.io/name=grafana --tail=50 확인 후 문제 없으면 3번 진행.
3. 왼쪽 메뉴 Explore 이동 -> 왼쪽 위 데이터 소스 드롭다운 Loki인지 확인.
4. Label filters의 Select label: namespace, Select value: monitoring 선택 -> Run query 클릭.
