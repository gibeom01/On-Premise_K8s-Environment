# ArgoCD Applications 설정

1. General (기본 설정)

Application Name: noblesys-monitoring
Project Name: default
SYNC POLICY: Automatic (체크 시 GitHub에 코드가 올라가면 즉시 K8s에 자동 반영됩니다.)
세부 옵션: Prune Resources(Git에서 파일 지우면 K8s에서도 삭제)와 Self Heal(누가 K8s에서 강제로 수정하면 Git 기준으로 원복)도 켜주시면 완벽한 GitOps가 됩니다.

2. Source (GitHub 저장소 설정)

Repository URL: GitHub 레포지토리 주소 (https://github.com/gibeom01/IDC_Physical_Server_K8s.git)
Revision: HEAD 또는 main (브랜치 이름)
Path: k8s-infra/monitoring (GitHub 레포지토리 내에서 sealed-secret.yaml과 cronjob.yaml이 들어있는 폴더 경로)

3. Destination (배포될 K8s 클러스터 위치)

Cluster URL: [https://kubernetes.default.svc](https://kubernetes.default.svc) (현재 ArgoCD가 설치된 내부 클러스터)
Namespace: monitoring (이전에 SealedSecret을 만들 때 지정했던 네임스페이스)

# 암호화 명령어 실행 및 확인

1. kubectl create secret generic grafana-auth-secret   --namespace monitoring   --from-literal=userKey="admin" --from-literal=passwordKey="admin-password"   --dry-run=client -o yaml | kubeseal --format=yaml > grafana-auth-secret.yaml
2. cat grafana-auth-secret.yaml
3. grafana-auth-secret.yaml의 userKey, passwordKey 암호화된 값 입력 후 github 업로드
