물리 서버 하드웨어 딥(Deep) 모니터링 (ipmi-exporter) 방법

1단계: 서버 BMC 설정 (사전 준비)
가장 먼저 모니터링 대상 서버의 iDRAC/iLO 설정에서 다음을 활성화해야 합니다.

IPMI over LAN 활성화.

전용 읽기 계정 생성 (예: ID prometheus / PW monitor123).

2단계: IPMI Exporter 설치 (Helm)
마스터 노드에서 실행합니다. 이 엑스포터는 서버 외부에서 관리 포트 IP로 접속하여 정보를 가져오는 '프록시' 역할을 합니다.

Bash
# 1. 차트 저장소 추가
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

# 2. ipmi-values.yaml 작성 (인증 정보 설정)
cat << 'EOF' > ipmi-values.yaml
modules:
  default:
    user: "prometheus"      # 💡 iDRAC/iLO에서 만든 계정
    pass: "monitor123"      # 💡 비밀번호
    driver: "LAN_2_0"
    privilege: "USER"
EOF

# 3. 설치
helm install ipmi-exporter prometheus-community/prometheus-ipmi-exporter \
  --namespace monitoring \
  -f ipmi-values.yaml

3단계: 프로메테우스 스크래핑 설정 추가
이미 설치된 kube-prometheus-stack이 외부 하드웨어 IP를 찔러보도록 monitoring-values.yaml 하단에 추가 설정을 넣고 업데이트해야 합니다.

YAML
# monitoring-values.yaml 에 추가
additionalServiceMonitors:
  - name: ipmi-exporter-targets
    jobLabel: hw-monitoring
    selector:
      matchLabels:
        app.kubernetes.io/name: prometheus-ipmi-exporter
    endpoints:
      - port: http
        interval: 30s
        path: /ipmi
        params:
          module: ["default"] # 💡 공통 계정(prometheus/monitor123)을 사용하는 모듈
          target: 
            - "192.168.1.10"  # Dell R640
            - "192.168.1.20"  # HPE Gen10
            - "192.168.1.30"  # Lenovo SR650

설정 후 업데이트:

Bash
helm upgrade monitoring ./kube-prometheus-stack-56.0.0.tgz \
  --namespace monitoring \
  -f monitoring-values.yaml

📊 3. Grafana 대시보드 연동
데이터가 들어오기 시작하면, 엔지니어님의 전공 분야인 Dell/HPE 서버 전용 대시보드를 입힐 차례입니다.

Grafana 접속 ➡️ Dashboards ➡️ New ➡️ Import

ID 11354 (IPMI Exporter 대시보드) 입력 후 Load.

데이터 소스를 Prometheus로 지정하고 저장합니다.
